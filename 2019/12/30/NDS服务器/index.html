把防火墙开启，在本机进行访问，看是否访问成功？
在客户端用地址访问网站能否成功？不能就关闭防火墙。
默认情况下，防火墙是不允许访问80端口的。

基于域名的访问（通过/etc/hosts文件设置的，只限本机有效）
在客户端计算机使用域名进行访问能成功吗？

DNS
正向解析：将域名解析成IP地址，A记录 CNAME记录 MX记录
反向解析：将IP地址解析成域名，PTR指针
DNS的查询方式：递归查询 迭代查询
区域类型：hint 根区域 master 主要区域 slave 辅助区域

配置DNS服务器：

建DNS服务器，分别以attendance和www.exam.com域名分别访问两个网站

1、安装DNS服务器

（1）安装bind程序包：

#yum install bind

（2）启动服务

# systemctl start named.service

（3）设置自启动状态

# systemctl enable named.service

2、主配置文件配置：

#vim /etc/named.conf

3、配置辅助区域文件的内容：

/etc/named.rfc1912.zones

4、配置区域配置文件：

将/var/named/named.empty复制重命名为exam.com.zone，

修改文件属组为named，并对其内容进行修改。

cp named.localhost exam.com.zone

chown :named exam.com.zone

5启动服务/重启服务

systemctl restart named.server

6设置DNS地址

7.测试DNS：nslookup

配置辅助DNS服务器

1）安装bind
2）启动bind
service named start
chkconfig named on
3）修改主配置文件，添加辅助区域
vim /etc/named.conf

zone "zlt.com" IN {
       type slave;
       file "slaves/slave.com.zone";
       masters {192.168.0.1;};
    };

zone "0.168.192.in-addr.arpa" IN {
       type slave;
       file "slaves/0.168.192.zone";
       masters {192.168.0.1;};
    };

4)重启服务：service named restart
